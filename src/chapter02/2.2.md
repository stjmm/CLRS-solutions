# 2-2
SELECTION-SORT(A,n):
    for i = 1 to n - 1:
        min = i
        for j = i + 1 to n
            if A[j] < A[i]
                min = j
        temp = A[i]
        A[i] = A[min]
        A[min] = temp;

**Loop invariant**
Init:
- At the start of each iteration of the outer loop A[1..i-1] contains the smallest i-1 elements of the array.
- The inner loop contains smallest number of A[i..j-1]
Maintnance:
- During iteration i algorithm find the smallest elem in unsorted array and swaps it into position A[i..n], so A[1..i] contains sorted array

**Why does it need to run for only the first n-1 elements?**
At the last step the algorithm will have 2 steps left to compare. It will store the larger one in A[n] and larger one in A[n-1]

**Worst case scenario**
(n - 1) + (n - 2) + .. 1 = (n)(n - 1)/2
So worst case scenario is O(n^2)

**Best case scenario**
Is the same as worst case scenario because it always has to scan through the unsorted array.
