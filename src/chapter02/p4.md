# b.

An array sorted decreasingly.
We can make up a simple algorithm to sum inversions in a sorted decreasingly array
SUM-INVERSIONS
for i = 1 to n - 1
    inversions += n - i
Or <n, n-1, ..., 1>, $\sum_{i=1}^n = (n - 1) + (n - 2) + ... + 1 = n/(n - 2)$

# c.

The running time is constant time the number of inversions. Let $I(i)$ denote the number of i < j such that A[j] > A[i]. $\sum_{j=1}^{n} I(i)$ are sum of inversions. The inner loop of insertionm sort will thus execute I(j) times, so insertion sort will execute $\sum_{i=1}^{n} I(j) times. Which is exactly the number of inversions in A.

# d.

SUM-INVERSIONS-MERGE(A, p, r)
    if p >= r
        return

    q = p + (r - p) // 2
    left = SUM-INVERSIONS-MERGE(A, p, q)
    right = SUM-INVERSIONS-MERGE(A, q + 1, r)
    inv = MERGE(A, p, q, r) + left + right
    return inv

MERGE(A, p, q, r)
    inv = 0

    nl = q - p + 1
    nr = r - q
    L[1..nl], R[1..nr]

    for i = 1 to nl
        L[i] = A[p + i - 1]
    for i = 1 to nr
        R[i] = A[q + i]

    i = 1
    j = 1
    k = p

    while i <= nl AND j <= nr
        if L[i] < R[j]
           A[k] = L[i] 
           i++
        else
            A[k] = R[j]
            j++
            inv += nl - i + 1
        k++

    while i <= nl
        A[k] = L[i]
        i++
        k++
    while j <= nr
        A[k] = R[j]
        j++
        k++
    
    return inv
