# a.

Insertion sort can sort each k element sublist in O(k^2) time.
So $n/k * O(k^2) = O(nk)$

---

# b.

1. We start with $n/k$ sublists of length $k$
2. We pair up sublists so we have $n/2k$, $n/4k$ until theres only one list

Proof:
1. Since we start with $m = n/k$ sublists there are $\lg(n/k)$ merges
2. Total work per each level is $n$
3. Final is O(n \lg(n/k))
EOP
